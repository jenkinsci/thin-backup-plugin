Falls diese Option aktiviert ist, werden die gematchten Daten ebenfalls zum Backup hinzugefügt, was größere Backups bedeutet!

Falls es zusätzliche Dateien gibt, die ins Backup aufgenommen werden sollen, so gibt es die Möglichkeit eine RegEx anzugeben. Alle Dateien mit einem Namen, der über die RegEx gematcht werden, werden dem Backup hinzugefügt. Falls der Ausdruck nicht valide ist, wird er ignoriert.
Beachte dies erlaubt nur Dateinamen Muster, (Keine ant-style patterns) aber positive und negative look-aheads sind effektiv um den primären Ordner herauszufiltern.
Beispiel:
^(email\-templates|.*\.jelly)$ wird email-templates/*.jelly herausfiltern.
^(?!jobs)(?!plugins)(?!logs)(?!users)(log|.*\.xml)$ wird *.xml und log/*.xml entahlten, wobei jobs, logs, users und plugins Ornder ausgenommen sind.
